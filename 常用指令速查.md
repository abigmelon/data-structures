# 📋 Git 常用指令速查

---

### 基本配置
```bash
git config --global user.name "你的名字"      # 设置用户名
git config --global user.email "你的邮箱"     # 设置邮箱
git config --list                             # 查看当前配置
```

### 创建/克隆仓库
```bash
git init                          # 初始化本地仓库
git clone <仓库URL>                # 克隆远程仓库到本地
```

### 日常提交流程
```bash
git status                        # 查看文件状态（哪些改了、哪些没暂存）
git add <文件名>                   # 暂存指定文件
git add .                         # 暂存所有改动
git commit -m "提交信息"           # 提交到本地仓库
git push origin main              # 推送到 GitHub
git push -u origin main           # 首次推送（建立追踪关系）
```

### 拉取与同步
```bash
git pull origin main              # 拉取远程最新代码并合并
git fetch                         # 只下载远程更新，不合并
```

### 分支操作
```bash
git branch                        # 查看所有分支
git branch <分支名>                # 创建新分支
git checkout <分支名>              # 切换到某分支
git checkout -b <分支名>           # 创建并切换到新分支
git merge <分支名>                 # 合并指定分支到当前分支
git branch -d <分支名>             # 删除分支
```

### 查看历史
```bash
git log                           # 查看提交历史
git log --oneline                 # 简洁版提交历史
git log --graph --oneline --all   # 图形化分支历史
git diff                          # 查看未暂存的改动
git diff --staged                 # 查看已暂存的改动
```

### 撤销操作
```bash
git reset HEAD <文件名>            # 取消暂存
git checkout -- <文件名>           # 撤销文件修改（慎用）
git reset --soft HEAD^            # 撤销 commit，保留修改
git reset --hard HEAD^            # 撤销 commit，丢弃修改（慎用）
git revert <commitID>             # 安全撤销（生成新 commit）
```

### 远程仓库
```bash
git remote -v                     # 查看关联的远程仓库
git remote add origin <URL>       # 关联远程仓库
git remote remove origin          # 取消关联
```

### 暂存工作区
```bash
git stash                         # 暂存当前修改
git stash pop                     # 恢复暂存的修改
git stash list                    # 查看暂存列表
```

### .gitignore 常用模板
```
*.exe
*.o
*.obj
.vscode/
Thumbs.db
Desktop.ini
```

### 典型工作流
```bash
git add .                           # 1. 暂存所有修改
git status                          # 2. 检查状态
git commit -m "完成第X周作业"        # 3. 提交
git push origin main                # 4. 推送到 GitHub
```
